# PET Center — Gestión de Milestones

## Propósito del documento

Este documento define los principales hitos (Milestones) del proyecto PET Center, estableciendo objetivos intermedios que permiten organizar el trabajo, monitorear el avance y validar el cumplimiento de entregables a lo largo del ciclo de desarrollo.

Cada Milestone agrupa un conjunto de Historias de Usuario (HU), Requerimientos No Funcionales (RNF) y tareas técnicas relacionadas, facilitando la planificación de los sprints y el seguimiento del proyecto mediante herramientas ágiles como GitHub Issues y GitHub Milestones.

Los milestones representan puntos de control clave que permiten verificar la evolución del proyecto desde la etapa inicial de análisis y diseño hasta la implementación completa y despliegue de la plataforma.

La definición de estos hitos facilita la gestión del alcance, la priorización de funcionalidades y el monitoreo del progreso del proyecto desde la etapa inicial de análisis y diseño hasta la integración y despliegue de la plataforma.

---

# 1. Estrategia de definición de Milestones

Para la planificación del proyecto se definieron tres milestones principales alineados con la metodología Scrumban adoptada por el equipo.

Cada Milestone representa un objetivo funcional concreto y agrupa Historias de Usuario (HU), Requerimientos No Funcionales (RNF) y tareas técnicas necesarias para alcanzar dicho objetivo.

Los milestones fueron definidos considerando:

* Dependencias funcionales entre módulos.
* Prioridad de negocio.
* Complejidad técnica.
* Distribución de trabajo por sprint.
* Objetivos establecidos en la planificación SMART.

# 2. Milestones del Proyecto

Con el objetivo de organizar el desarrollo de PET Center en etapas claramente definidas, se establecieron tres milestones principales alineados con la planificación de sprints y los objetivos del proyecto.

Cada Milestone representa un conjunto de funcionalidades y entregables que permiten alcanzar un resultado concreto y verificable dentro del ciclo de vida del proyecto. La finalización de cada hito constituye un punto de control que facilita el seguimiento del avance, la validación de requerimientos y la gestión de riesgos.

Los milestones fueron definidos considerando:

* La prioridad de las funcionalidades para el producto mínimo viable (MVP).
* Las dependencias técnicas entre módulos.
* La distribución del trabajo en los distintos sprints.
* Los objetivos establecidos en el documento SMART.
* Las necesidades identificadas durante el análisis de stakeholders y requerimientos.

A continuación, se describen los tres hitos principales definidos para el proyecto PET Center.

## 2.1 M1 — Base del Proyecto

Este Milestone concentra las actividades necesarias para establecer las bases funcionales y técnicas del proyecto. Incluye la documentación inicial, la definición metodológica, la configuración del entorno de desarrollo y las primeras funcionalidades orientadas al registro y autenticación de usuarios.

El objetivo principal es disponer de una plataforma inicial que permita a los usuarios registrarse, iniciar sesión y comenzar a explorar los servicios disponibles dentro del ecosistema PET Center.

## 2.2 M2 — Núcleo Funcional

Este Milestone agrupa las funcionalidades centrales del negocio relacionadas con la gestión de mascotas, la reserva de servicios y la interacción entre clientes y proveedores.

Durante esta etapa se implementan los procesos principales que aportan valor al usuario final, incluyendo la disponibilidad de servicios, la administración de reservas y el sistema de valoraciones.

La finalización de este hito permite contar con una plataforma operativa capaz de gestionar el flujo principal de uso previsto para PET Center.

## 2.3 M3 — Plataforma Completa

Este Milestone contempla la consolidación de la plataforma mediante la incorporación de funcionalidades administrativas, pruebas integrales, actividades de aseguramiento de calidad y preparación para el despliegue final.

Además de las funcionalidades de gestión, se incluyen los requerimientos no funcionales relacionados con disponibilidad, rendimiento, compatibilidad y experiencia de usuario.

La finalización de este hito representa la entrega de una versión completa de PET Center lista para ser publicada y utilizada por sus usuarios.

# 3. Implementación y Seguimiento

Los milestones definidos en este documento serán gestionados mediante las herramientas de planificación y seguimiento adoptadas para el proyecto, permitiendo mantener la trazabilidad entre los requerimientos, las tareas de desarrollo y los entregables finales.

Cada milestone agrupa un conjunto de Historias de Usuario (HU), Requerimientos No Funcionales (RNF) y tareas técnicas asociadas a un objetivo específico. Estas actividades son registradas y monitoreadas mediante el Product Backlog, el tablero de seguimiento y los mecanismos de control de versiones utilizados por el equipo.

La utilización de milestones permite:

* Organizar el trabajo en etapas claramente definidas.
* Visualizar el progreso del proyecto de forma incremental.
* Facilitar el seguimiento de entregables y dependencias.
* Evaluar el cumplimiento de objetivos parciales.
* Mantener la alineación entre planificación y ejecución.

Durante el desarrollo, cada Historia de Usuario y tarea asociada podrá vincularse al Milestone correspondiente, permitiendo conocer en todo momento el estado de avance de cada etapa del proyecto.
